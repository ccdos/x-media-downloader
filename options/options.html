X Media Downloader Settings

Customize how downloads are saved, plus filename templates and an optional Downloads subfolder. By default, filenames do not include {author} or {tweetId}; those fields only appear when you add them to your template.

Download behavior Use the saved Downloads subfolder for one-click downloads, or switch to Always ask where to save when you want Chrome to open the save location picker for each file.

Default settings

  • Download behavior: Use saved Downloads subfolder
  • Downloads subfolder: (empty)
  • Primary template: x_{postTitle}_{kind}_{index}.{ext}
  • Fallback: x_{kind}_{index}.{ext}

Available fields

FieldMeaning
{postTitle}A short title extracted from the visible post text. Whitespace and unsupported filename characters are normalized automatically. Empty when no usable post text is available.
{tweetId}The numeric post ID.
{kind}The media type, usually image or video.
{index}The 1-based media index within the current post.
{ext}The resolved file extension, for example jpg, png, or mp4.
{author}The post author's handle. It is excluded from the default templates and only appears if you add it yourself.

Notes

  • The Downloads subfolder is relative to your browser Downloads folder. Chrome blocks absolute paths and backtracking such as .. in extension-initiated downloads.
  • When Download behavior is set to Always ask where to save, Chrome opens its save location picker for every file and the saved Downloads subfolder is ignored.
  • You can mix plain text with fields, for example: downloads/{tweetId}_{kind}_{index}.{ext}
  • If the primary template contains {postTitle} but the current post does not expose usable text, the extension automatically falls back to the fallback template.
  • Unsupported filename characters in the rendered template are sanitized automatically.
